# تقرير التعديلات على نظام المهام

## الملخص العام
تم إضافة مميزات جديدة شاملة لنظام المهام والتقييمات اليومية بهدف تحسين تتبع أداء الأطفال وتقديم ملاحظات مفصلة من الآباء.

## 1️⃣ التعديلات على نموذج المهام (Task Model)

### الحقول الجديدة المضافة:
- **`category`** (التصنيف): تصنيف المهمة مثل:
  - Daily Chores (أعمال يومية)
  - Culture (ثقافة)
  - Financial (مالية)
  - Religion (دينية)

- **`is_auto_verified`** (التحقق التلقائي): قيمة منطقية (True/False)

- **`verification_type`** (نوع التحقق):
  - AUTO: التحقق التلقائي
  - MANUAL: التحقق اليدوي

### الحقول الموجودة:
- `title`: اسم المهمة
- `description`: وصف المهمة
- `child_id`: معرف الطفل
- `points`: نقاط المكافأة
- `status`: حالة المهمة (PENDING, APPROVED, REJECTED)
- `task_type`: نوع التكرار (ONCE, DAILY, WEEKLY)
- `recurrence_day`: يوم التكرار الأسبوعي (0-6)

---

## 2️⃣ نموذج التقييمات اليومية (DailyFeedback)

تم إنشاء نموذج جديد يحتوي على:

### الحقول:
- **`child_id`**: معرف الطفل
- **`feedback_date`**: التاريخ (يوم واحد فقط لكل طفل)
- **`emoji_value`**: قيمة التقييم:
  - 1 = 😊 سعيد (أداء ممتاز)
  - 2 = 😐 محايد (أداء جيد مع نقاط)
  - 3 = 😢 حزين (أداء غير مرضي)
- **`feedback_text`**: تعليق اختياري
- **`created_by`**: معرف الوالد
- **`created_at`**: تاريخ الإضافة

### ملاحظة مهمة:
التقييم يكون **يومياً واحداً فقط** لكل طفل، وليس لكل مهمة على حدة.

---

## 3️⃣ نقاط نهاية المهام والتقييمات

### نقاط نهاية المهام:
- `POST /api/tasks/` - إنشاء مهمة
- `PUT /api/tasks/{task_id}` - تحديث مهمة

### نقاط نهاية التقييمات:
- `POST /api/tasks/feedback/` - إنشاء تقييم يومي جديد
- `GET /api/tasks/feedback/{feedback_id}` - جلب تقييم معين
- `PUT /api/tasks/feedback/{feedback_id}` - تحديث التقييم
- `DELETE /api/tasks/feedback/{feedback_id}` - حذف التقييم
- `GET /api/tasks/feedback/child/{child_id}` - جميع التقييمات
- `GET /api/tasks/feedback/child/{child_id}/{date}` - تقييم تاريخ محدد

---

# نظام النقاط والمكافآت

## نموذج النقاط (Point Model) ⭐

تم إنشاء نموذج جديد `Point.py` لتتبع النقاط اليومية.

### الحقول:
- **`child_id`**: معرف الطفل
- **`task_id`**: معرف المهمة (اختياري)
- **`points_earned`**: عدد النقاط المكتسبة
- **`points_set_by_parent`**: قيمة النقاط (1-5 نقاط)
- **`earning_date`**: تاريخ كسب النقاط (يومي)
- **`created_by`**: معرف الوالد
- **`notes`**: ملاحظات

### الدوال الرئيسية:
- `get_daily_points()` - إجمالي النقاط اليومية
- `get_weekly_points()` - إجمالي النقاط الأسبوعية
- `get_points_history()` - سجل 30 يوم

### ملاحظات:
- كل مهمة = **5 نقاط (افتراضي)**
- الوالد يعدل القيمة (1-5 نقاط)
- النقاط **يومية التتبع**
- منفصلة عن المكافآت

---

## نموذج المكافآت (Reward Model) 🎁

تم إنشاء نموذج جديد `Reward.py` لإدارة المكافآت الأسبوعية.

### الحقول:
- **`child_id`**: معرف الطفل
- **`reward_name`**: اسم المكافأة
- **`description`**: وصف المكافأة
- **`cost_in_points`**: تكلفة بالنقاط (اختياري)
- **`reward_value`**: قيمة إضافية (مثل: "ساعة واحدة")
- **`reward_type`**: نوع المكافأة:
  - TIME_BASED: وقت إضافي
  - OUTING: نشاط خارجي
  - CASH: مكافأة مالية
  - ENTERTAINMENT: ترفيه
  - EXPERIENCE: تجربة
- **`is_active`**: حالة المكافأة (on/off)
- **`week_start_date`**: بداية الأسبوع (الخميس)
- **`week_end_date`**: نهاية الأسبوع (الأربعاء)

### المكافآت الافتراضية:
1. ⏱️ وقت إضافي - بلايستيشن (ساعة واحدة)
2. ⏱️ وقت إضافي - ألعاب (30 دقيقة)
3. 🎬 سينما (تذكرتان)
4. 👥 زيارة أصدقاء (ساعتان)
5. 💰 مكافأة مالية (20 ريال)
6. ⭐ أخرى (مكافأة من اختيار الوالد)

### الدوال الرئيسية:
- `get_current_week_rewards()` - جميع مكافآت الأسبوع
- `get_active_rewards()` - المكافآت المفعلة (on)
- `toggle_reward()` - تفعيل/تعطيل
- `create_default_rewards()` - المكافآت الافتراضية

### ملاحظات مهمة:
- **كل خميس**: تتجدد المكافآت تلقائياً
- المكافآت **منفصلة تماماً عن النقاط**
- الوالد يختار المكافآت (on/off)
- 6 مكافآت افتراضية ثابتة

## خدمات النقاط والمكافآت (Services) 🔧

تم إنشاء ملف `reward_service.py` بـ 3 خدمات رئيسية:

### 1️⃣ **PointService**:
```
- add_points() - إضافة نقاط
- get_daily_points_summary() - ملخص يومي
- get_weekly_points_summary() - ملخص أسبوعي
- get_points_history() - سجل النقاط
- update_point_value() - تحديث القيمة
```

### 2️⃣ **RewardService**:
```
- get_week_rewards() - جميع المكافآت
- get_active_rewards() - المكافآت المفعلة
- toggle_reward_status() - تبديل الحالة
- create_week_rewards() - إنشاء أسبوعي
```

### 3️⃣ **DailyProgressService**:
```
- get_daily_progress() - تقدم يومي
- get_progress_chart_data() - بيانات الرسم البياني
- get_weekly_progress_report() - تقرير أسبوعي
```

# دليل الاستخدام الشامل

## 1️⃣ نظام النقاط

### البيانات الأساسية:
- كل مهمة تعطي **5 نقاط (افتراضي)**
- الوالد يعدل القيمة (1-5 نقاط إضافية)
- النقاط تتراكم يومياً
- النقاط مستقلة عن المكافآت

### المسارات (Endpoints):

#### جلب النقاط اليومية
```bash
GET /api/dashboard/points/daily/child-123
```

#### جلب النقاط الأسبوعية
```bash
GET /api/dashboard/points/weekly/child-123
```

#### جلب سجل النقاط
```bash
GET /api/dashboard/points/history/child-123?days=30
```

#### إضافة نقاط
```bash
POST /api/dashboard/points/add
{
  "child_id": "child-123",
  "points_earned": 10,
  "task_id": "task-456",
  "parent_id": "parent-789",
  "points_set_by_parent": 5,
  "notes": "أكمل المهمة بنجاح"
}
```

#### تحديث نقطة
```bash
PUT /api/dashboard/points/update/point-1
{
  "new_points_value": 15
}
```

## 2️⃣ نظام المكافآت

### البيانات الأساسية:
- المكافآت تتجدد **كل خميس**
- الوالد يختار المكافآت (on/off)
- **6 مكافآت افتراضية ثابتة**
- لا توجد مكافآت مخصصة

### المسارات (Endpoints):

#### جلب جميع مكافآت الأسبوع
```bash
GET /api/dashboard/rewards/week/child-123
```

#### جلب المكافآت المفعلة فقط
```bash
GET /api/dashboard/rewards/active/child-123
```

#### تفعيل/تعطيل مكافأة
```bash
PUT /api/dashboard/rewards/toggle/reward-1
```

#### إنشاء مكافآت أسبوعية افتراضية
```bash
POST /api/dashboard/rewards/create-weekly/child-123/parent-789
```

## 3️⃣ نظام التقدم اليومي والرسم البياني

```bash
GET /api/dashboard/progress/daily/child-123?date=2026-06-04
```

#### جلب تقرير الأسبوع
```bash
GET /api/dashboard/progress/weekly-report/child-123
```
