# Sprint Planning

## Purpose

Sprint Planning was conducted before the start of each sprint to define the sprint goal, identify the features to be implemented, prioritize tasks, assign responsibilities among team members, and estimate the work required to complete the sprint successfully.

---

# Sprint 1 Planning

## Sprint Goal

Establish the project's foundation by implementing authentication, child management, and preparing the core backend and frontend structure.

## Sprint Scope

### In Scope

* Backend architecture.
* Database models and relationships.
* Parent registration and login APIs.
* Child login API.
* Child management APIs.
* Parent authentication screens.
* Child login screen.
* Parent dashboard.
* Add Child screen.

### Out of Scope

* Task management.
* Wishlist and rewards features.
* Final testing and deployment.

## Task Assignment

* **Backend Developer**

  * Develop backend architecture.
  * Implement database models.
  * Implement authentication APIs.
  * Implement child management APIs.
  * Maintain API documentation.

* **Frontend Team**

  * Prepare Flutter project structure.
  * Build authentication screens.
  * Build Parent Dashboard.
  * Build Add Child screen.
  * Prepare API integration structure.

## Sprint Schedule

The team conducted weekly planning meetings in person and through Google Meet to discuss sprint goals, assign tasks, share technical knowledge, and resolve blockers. GitHub was used as the main source repository for collaboration and version control.

## Sprint Scope

### In Scope

* Task editing and deletion API.
* Task assignment API.
* Task completion API.
* Add Task screen.
* Child Tasks screen.
* API integration between frontend and backend.

### Out of Scope

* Wishlist features.

* Advanced improvements.

## Task Assignment

* **Backend Developer**

  * Implement task management APIs.
  * Handle task assignment logic.
  * Implement task completion workflow.
  * Fix API issues during integration.

* **Frontend Team**

  * Build task-related screens.
  * Connect Flutter screens with backend services.
  * Implement task workflow UI.
  * Test frontend behavior.

## Sprint Schedule

The team focused on completing the complete task workflow, from task creation and assignment until the child receives and completes assigned tasks.

# Sprint 4 Planning

## Sprint Goal

Complete additional MVP features, perform final testing, and prepare the application for deployment.

## Sprint Scope

### In Scope

* Wishlist feature.
* Wishlist approval.
* Wishlist goals.
* Weekly rewards.
* Weekly reward approval.
* Final integration testing.
* Bug fixing.
* Performance improvements.

### Out of Scope

* Language Switching (Arabic ↔ English).
* Progress Star (Star Growth System).
* Emoji Feedback.
* Full Analytics Dashboard.

## Task Assignment

* **Backend Developer**

  * Implement Wishlist APIs.
  * Implement Rewards APIs.
  * Fix backend issues.
  * Improve API stability.

* **Frontend Team**

  * Build Wishlist screen.
  * Build Weekly Rewards screen.
  * Complete UI improvements.
  * Verify full application workflow.

* **QA**

  * Perform final integration testing.
  * Validate API and UI behavior.
  * Report and verify bug fixes.

## Sprint Schedule

The final sprint focused on completing remaining MVP features, validating the complete workflow, fixing issues, and preparing the application for final demonstration.

---

# Sprint Collaboration

Weekly meetings were conducted both in person and through Google Meet. Team members discussed progress, technical challenges, blockers, and upcoming tasks. Collaboration was managed through GitHub, where branches, commits, and code changes were reviewed and maintained.

---

# Overall Sprint Outcome

The project was developed incrementally through four one-week sprints. Each sprint delivered a specific part of the MVP, starting from authentication and child management, progressing through task workflows and approval systems, and ending with wishlist, rewards, testing, and final improvements.
